# Changelog index

pypdfbox keeps its changelog and release notes at the repository
root. Use this page as the dispatcher.

- **[`CHANGES.md`](../CHANGES.md)** — *active behavioural
  divergences* from upstream Apache PDFBox, plus the delta log.
  Read this when you want to know how the Python port
  deliberately deviates from the Java reference, or what landed
  most recently.
- **[`RELEASE_NOTES_v0.9.0rc1.md`](../RELEASE_NOTES_v0.9.0rc1.md)**
  — *release-specific notes* for the 0.9.0rc1 cut. Status, major
  features, active divergences, migration pointer, known open
  items.

For *open follow-up items* (fixable, not yet done) see the
[issue tracker](https://github.com/mehdyhaghy/pypdfbox/issues).
For *contribution rules* see
[`CONTRIBUTING.md`](../CONTRIBUTING.md). For *per-file porting
attribution* see [`PROVENANCE.md`](../PROVENANCE.md).

Routine mechanical translations (`camelCase → snake_case` and
class-by-class porting without behavioural deviation) are not
recorded in the changelog — they show up in git history and
`PROVENANCE.md` only.
